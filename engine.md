## 1. 架构总览

```
┌──────────┐        ┌─────────────────┐        ┌──────────────┐
│  Sensor  │ a, ω   │  ThrowDetector  │ v0,    │  Simulator   │
│ (平台API) │ ─────► │  (状态机+出手解算) │ pitch, │  (气动力积分)  │
│  ≥60Hz   │        │                 │ roll   │              │
└──────────┘        └─────────────────┘        └──────────────┘
                         事件触发                 每帧 ≥60Hz
```

| 模块 | 输入 | 输出 | 职责 |
|---|---|---|---|
| Sensor | 平台传感器 API | `a` (m/s²), `ω` (rad/s) | 原始数据采集、频率监控 |
| ThrowDetector | `a, ω` 时间序列 | `v0, pitch, roll, fwd, up` | 状态机 + 握姿校验 + 出手识别 + 初始姿态解算 |
| Simulator | `v0, fwd, up` + 物理参数 | 轨迹、距离、高度、时长、侧偏 | 升力/阻力/侧力/重力 四力积分 |

**核心设计决定**：

- **不用通用物理引擎的刚体**（Box2D/Bullet/PhysX）。纸飞机的升力/阻力/失速需要自定义气动力模型，通用引擎不解算这些。
- **出手姿态用互补滤波**：陀螺仪高频积分 + 加速度计低频修正，既避免纯陀螺漂移，又捕获挥动过程中的姿态变化。
- **飞行中机身姿态非固定**：引入被动机身对齐（attitudeTau 时间常数），模拟纸飞机重心-气压中心偏置带来的自动俯仰修正。

## 2. 坐标系定义

整个系统涉及 3 个右手坐标系。

### 2.1 世界系 (World)

地面为 XY 平面，天空为 +Z。

```
        +Z (天空)
         │
         │
         └────── +X (玩家右侧)
        ╱
      +Y (飞行前进方向)
```

| 轴 | 方向 | 物理含义 |
|---|---|---|
| +X | 右 | 玩家右手方向，侧偏度量 |
| +Y | 前 | 飞行前进方向，距离度量 |
| +Z | 上 | 竖直向上，高度度量 |

**基准线**：从原点沿 +Y 延伸的射线，带距离刻度。落地点到基准线的垂足刻度即为飞行距离。

**Cocos Creator 差异**：Cocos 3.x 默认世界系为 `+X右, +Y上, -Z前`。迁移时需做坐标映射（详见 §11）。

### 2.2 手机本体系 (Phone)

W3C DeviceMotion / 微信 `wx.onAccelerometerChange` 标准：

```
        +Y (机顶 / 听筒侧)
         │
         │
         └────── +X (机右 / 物理按键侧)
        ╱
      +Z (屏幕外 / 出屏方向)
```

- `+X`：机右（物理按键一侧）
- `+Y`：机顶（听筒/前置摄像头一侧）
- `+Z`：屏幕外（垂直屏幕指向用户）

### 2.3 机身系 (Body)

物理仿真用，右手系：

- `X_body`：机头方向（前向），世界系下记为 `fwd`
- `Y_body`：机背方向（上方），世界系下记为 `up`
- `Z_body`：右翼方向，`= fwd × up`（不显式存储，需要时计算）

`fwd` 与 `up` 始终保持正交且单位长度，由发射瞬间初始化，飞行中通过被动对齐更新。

### 2.4 手机系 → 世界系 变换

标准握姿（MRD §3.1 规定）的物理对应：

| 世界方向 | 手机本体系 |
|---|---|
| 前 (+Y_w) | 手机 -Y（机底/充电口） |
| 上 (+Z_w) | 手机 -X（机左边缘） |
| 右 (+X_w) | 手机 -Z（屏幕背面） |

写成旋转矩阵 R（列 = 手机各轴在世界系下的方向向量）：

```
R = [[ 0,  0, -1],     ← +X_w = 手机 -Z
     [ 0, -1,  0],     ← +Y_w = 手机 -Y
     [-1,  0,  0]]     ← +Z_w = 手机 -X
```

**变换函数**：`worldVec = (-phoneVec.z, -phoneVec.y, -phoneVec.x)`

**等价四元数**（单位化）：`q_canon = (1/√2, 0, -1/√2, 0)`（分量顺序：x, y, z, w）

**握姿验证的物理依据**：标准握姿下世界重力 `g_w = (0, 0, -g)`，反变换到手机系得 `g_phone = (g, 0, 0)`（重力指向机右，机右朝下）。W3C 加速度计读数（比力 = -g）应为 `(-g, 0, 0)`。

### 2.5 手机系 → 机身系 变换

机身系定义与手机系不同：机身 X=前(机头), Y=上(机背), Z=右(右翼)。

在标准握姿下，手机 -Y（机底）→ 机身 +X（机头），手机 -X（机左边缘）→ 机身 +Y（机背），手机 -Z（屏幕背面）→ 机身 +Z（右翼）。

此变换由设备四元数 `q_device`（手机→世界）经固定旋转得到机身四元数 `q_body`（机身→世界）：

```
q_body = q_device · q_device_to_body
```

其中 `q_device_to_body` 为 180° 绕 (1,-1,0)/√2 轴的旋转（四元数：1/√2, -1/√2, 0, 0），等价于旋转矩阵 `[[0,-1,0],[-1,0,0],[0,0,-1]]`。

## 3. 传感器接口

### 3.1 数据格式

| 字段 | 类型 | 单位 | 说明 |
|---|---|---|---|
| `a` | Vec3 | m/s² | 加速度计读数（含重力，W3C 约定 = 比力） |
| `ω` | Vec3 | rad/s | 陀螺仪角速度（手机本体系） |
| `dt` | scalar | s | 距上次事件的采样间隔 |

### 3.2 平台适配

| 平台 | 加速度计 API | 陀螺仪 API | 备注 |
|---|---|---|---|
| 浏览器 | `DeviceMotionEvent.accelerationIncludingGravity` | `DeviceMotionEvent.rotationRate` | iOS 需主动请求权限 |
| 微信小游戏 | `wx.onAccelerometerChange` | `wx.onGyroscopeChange` | 频率参数设为 `game`（≥60Hz） |

### 3.3 轴适配参数

不同设备/平台可能存在轴映射和符号差异，需提供以下可配置项：

| 参数 | 默认值 | 说明 |
|---|---|---|
| `accelSwapYZ` | false | 加速度计 Y/Z 轴交换 |
| `gyroSwapYZ` | false | 陀螺仪 Y/Z 轴交换 |
| `gyroSignX` | +1 | 陀螺 X（beta）符号翻转 |
| `gyroSignY` | +1 | 陀螺 Y（gamma）符号翻转 |
| `gyroSignZ` | -1 | 陀螺 Z（alpha）符号翻转（W3C +alpha=顺时针 → 翻转） |

## 4. 投掷检测状态机

### 4.1 状态流转

```
IDLE ──[arm()]──► ARMED ──[校准通过]──► SWINGING ──[检测到出手峰]──► RELEASED
  ▲                  │                       │
  │                  │ 超时/握姿失败          │ 超时(>3s)
  └──────────────────┴───────────────────────┘
```

| 状态 | 进入条件 | 行为 | 退出条件 |
|---|---|---|---|
| IDLE | 初始 / reset() | 等待用户触发 | 用户调用 arm() |
| ARMED | arm() 调用 | 收集 200ms 静止样本，验证握姿 | 方向+静止性通过 → SWINGING；失败则滚动重试 |
| SWINGING | 校准成功 | 互补滤波追踪姿态，滚动窗追踪前向加速度 | 检测到出手峰 → RELEASED；超时 → ARMED |
| RELEASED | 出手检测触发 | 解算 v0/pitch/roll，触发反馈，启动仿真 | 一次性，Simulator 接管 |

### 4.2 ARMED 阶段：握姿校验

收集最近 200ms 的加速度计样本，计算均值 `a_mean` 和标准差 σ。

**方向判据**（单轴阈值）：

```
a_mean.x < -poseMain × g        // 主轴：机右朝下，a.x 应接近 -g
|a_mean.y| <  poseTol × g       // 余轴容差
|a_mean.z| <  poseTol × g       // 余轴容差
```

| 参数 | 默认值 | 含义 |
|---|---|---|
| `poseMain` | 0.7 | 主轴分量下限（0.7g → 容许约 45° 偏差） |
| `poseTol` | 0.4 | 余轴分量上限（0.4g → 容许约 24° 偏差） |

**静止性判据**（新增加）：

```
σ < poseStill × g               // 样本标准差 < 0.15g
```

防止持续晃动时通过校验。

### 4.3 SWINGING 阶段：姿态追踪

采用**互补滤波**（Android SensorManager 风格）：陀螺仪高频积分 + 加速度计低频修正。

**每帧流程**：

1. 陀螺低通滤波（平滑原始角速度）
2. 四元数积分：`q ← q · exp(ω·dt/2)`（指数映射闭式解）
3. 加速度计修正：当 `|a| ≈ g`（手机静止）时，用加速度计反推的姿态向陀螺积分结果做 lerp 混合
4. 前向加速度反推：用 q 将手机系加速度旋到世界系，取 +Y 分量得 `a_fwd`
5. 维护滚动窗口（默认 1500ms），追踪 `a_fwd` 时间序列

**四元数积分（指数映射）**：

```
给定 ω (rad/s) 和 dt (s)：
  half = ω · dt / 2
  θ = |half|
  dq = (half · sin(θ)/θ, cos(θ))    // θ→0 时 sin(θ)/θ→1
  q_new = normalise(q · dq)
```

比一阶加法精确，对常 ω 在 dt 内为闭式精确解。

### 4.4 出手检测

**候选峰检测**（3 样本局部最大）：

```
A = swing[n-3], B = swing[n-2], C = swing[n-1]
if B.a_fwd > triggerAccel
   AND B.a_fwd ≥ A.a_fwd
   AND B.a_fwd > C.a_fwd
   AND B.|ω| > omegaMin:
       candidatePeak = B
```

| 参数 | 默认值 | 说明 |
|---|---|---|
| `triggerAccel` | 15 m/s² | 前向加速度阈值（普通挥手 15-25，猛甩 30+） |
| `omegaMin` | 2.0 rad/s | 角速度下限，排除纯线性振动 |

**Lookahead 防首峰截胡**：检测到候选峰后等待 `lookaheadMs`（默认 80ms），期间若有更高 `a_fwd` 样本则更新候选峰；到期后提交真正的峰值。

## 5. 出手参数解算

### 5.1 初速 v0

从挥动起点到峰值时刻，对线性加速度向量积分：

```
v_vec = Σ (a_lin_world[i] + a_lin_world[i-1]) / 2 × dt_i
v0 = |v_vec| × kImpulse
```

**起点回溯**：从峰值位置向前找到第一个 `a_fwd < max(0.5, peak_a_fwd × 0.1)` 的样本。

**拒绝规则**：
- `v0 < vMin`（默认 3 m/s）：提示"力度不足"，退回 ARMED
- `v0 > vMax`（默认 25 m/s）：按 vMax 出手（软上限 + 警告）

### 5.2 出手姿态

用 Release 时刻的设备四元数 `q_release`（已通过互补滤波累积），经设备→机身固定旋转得到机身四元数：

```
q_body = normalise(q_release · q_device_to_body)
```

从 `q_body` 导出世界系下的机身轴：

```
fwd  = q_body · (1, 0, 0)_body    // 机头方向
up   = q_body · (0, 1, 0)_body    // 机背方向
```

反算 pitch / roll：

```
pitch = asin(fwd.z)                // fwd.z > 0 → 机头高于水平面
roll  = asin(-right.z)             // right = fwd × up
```

**边界检查**：
- `|pitch| > 60°` → 拒绝，退回 ARMED
- `|roll| > rollMax`（默认 45°）→ 拒绝，退回 ARMED

### 5.3 反馈

出手瞬间触发：
- 振动反馈（`navigator.vibrate(60)` 或微信 `wx.vibrateShort`）
- 屏幕闪白（30ms）+ 短促音效（880→440Hz 下降音）作为兜底反馈

## 6. 气动模型

### 6.1 体轴系速度分解

将世界速度 `v` 投影到机身三轴：

```
right = normalise(fwd × up)        // 右翼方向
v_fwd    = v · fwd                 // 纵向分量
v_normal = v · up                  // 法向分量（俯仰面内）
v_lat    = v · right               // 横向分量（偏航面内）
```

迎角 α 与侧滑角 β（均带符号）：

```
α = atan2(-v_normal, v_fwd)       // α>0 → 速度在机身下方 → 正迎角
β = atan2( v_lat,    v_fwd)       // β>0 → 速度在机身右侧 → 正侧滑
```

### 6.2 升力系数 CL

采用 **Hoerner 平板模型（含展弦比修正）**，统一公式无需分段：

```
CL(α) = a0 × sin(α) × cos(α) / (1 + 0.7 × |sin(α)|)
```

其中 `a0 = 2π / (1 + 2/AR)`，展弦比 AR 取 3.5（纸飞机典型值），`a0 ≈ 4.0`。

**物理背景**：纸飞机实质是一张平板，流体在板两侧产生压差。薄翼理论 `CL = 2π·α` 对平板在小 α 段高估约 3 倍。Hoerner 公式 `sin(2α)` 是平板的正确基准，加 AR 修正后更接近真实纸飞机。

**特性**：
- 小 α 处斜率 ≈ 4.0/rad（vs 旧 2π·α 的 6.28/rad，降低 ~36%）
- CL_max ≈ 1.34 出现在 α ≈ 45°
- 无突变失速（平板特性），CL 平滑过渡
- 参数 `alphaStall`、`CLpostStall` 不再需要

### 6.3 阻力系数 CD

```
CD(α, CL) = CD0_eff + k × CL² + sin²(α)
```

| 项 | 物理来源 | 说明 |
|---|---|---|
| `CD0_eff` | 零升阻力 | 表面摩擦 + 形阻，软下限 0.03 |
| `k × CL²` | 诱导阻力 | 产生升力的代价，k ≈ 1/(π·AR·e) |
| `sin²(α)` | 钝体压差阻力 | 大 α 时主导（"开车窗"效应），是失速俯冲的物理根源 |

**软下限**：当 `CD0 < 0.03` 时，`CD0_eff = 0.03 - (0.03 - CD0)² / 0.03`（二次过渡到 0.03）。

### 6.4 侧力系数 CY

```
CY(β) = -k_β × β
```

| 参数 | 默认值 | 说明 |
|---|---|---|
| `k_β` | 1.0 | 侧滑稳定性系数 |

**物理含义**：机身侧滑时产生与 β 反向的力（类似垂直尾翼的气象风向标效应），自然抑制无限侧偏。

### 6.5 力的合成

```
动压:        q = ½ × ρ × |v|²
升力:        F_lift  = q × S × CL × liftDir
侧力:        F_side  = q × S × CY × right
阻力:        F_drag  = -q × S × CD × v_hat
重力:        F_gravity = (0, 0, -m × g)
合力:        F = F_lift + F_side + F_drag + F_gravity
```

其中 `liftDir` 垂直于 v，且位于 (v, up) 张成的平面内：

```
inner   = up × v_hat
liftDir = normalise(v_hat × inner)
```

直观理解：升力沿机背方向推，但被速度方向"拉斜"——速度有多斜（pitch up/down），升力方向就斜多少。**这是横滚导致侧偏的根本原因**：横滚把 `up` 倾斜，升力方向跟着倾斜，水平分量推飞机侧向移动。

## 7. 运动方程与积分

### 7.1 Semi-Implicit Euler

```
F = F_lift + F_side + F_drag + F_gravity
a = F / m
v_new = v + a × dt      // 先更新速度
p_new = p + v_new × dt   // 用新速度更新位置
t += dt
```

比显式 Euler 更稳定，对周期运动（phugoid）能量误差小得多。

### 7.2 动态子步

当升力远大于重力时（L/W 极大），单步 Δv 可能与 v 同量级，导致数值发散。在各帧起始估算所需子步数：

```
q_est    = ½ × ρ × |v|²
CL_max   = 1.34（平板模型理论最大值）
F_l_max  = q_est × S × CL_max
a_net    = max(0, F_l_max / m - g)
subs_req = ceil(a_net × dt / (0.3 × |v|))    // 要求 Δv/v < 0.3
subs     = min(64, max(baseSubs, subs_req))
```

要求 `Δv_per_substep < 0.3 × v` 确保 semi-implicit Euler 稳定性。正常参数下子步数不变，仅极端条件（L/W 极大）时自动增加，上限 64。

### 7.3 帧率独立性

基础子步数 `baseSubs = min(maxSubsteps, ceil(dt / (1/120)))`，确保 60fps 下至少 2 子步/帧（等效 120Hz 积分），与显示帧率解耦。`maxSubsteps` 默认 4。

## 8. 被动机身对齐

### 8.1 动机

如果机身姿态完全锁死（fwd 永不变），飞机会进入经典 phugoid 长周期振荡：
下落→v 偏向下→α 增大→CL 飙升→升力远超重力→弹回去→速度损失→又掉下来→循环。

真实纸飞机有一种"自动俯仰修正"机制抑制此振荡：**重心稍位于气压中心之前**，当 α 变大时机身被气流推得低头，α 变小时被推得抬头。这部分用一个时间常数 `attitudeTau` 来模拟。

### 8.2 实现

在每个子步把 fwd（连同 up）绕 `fwd × v_hat` 轴向速度方向旋转一小步：

```
ang     = acos(fwd · v_hat)            // 当前 fwd 与速度的夹角
blend   = min(1, dt / attitudeTau)     // 本步对齐比例
rot_ang = ang × blend                  // 本步实际旋转角
axis    = normalise(fwd × v_hat)       // 旋转轴
rq      = 轴角 → 四元数(axis, rot_ang)
fwd_new = normalise(rq · fwd)
up_new  = normalise(rq · up)
```

### 8.3 参数

| 参数 | 默认值 | 效果 |
|---|---|---|
| `attitudeTau` | 0.3s | 经过 τ 时间，fwd 与 v_hat 夹角降至初始的 37% |

- τ 越小（~0.05s）：fwd 紧贴速度方向，phugoid 完全压平 → 轨迹平滑但"飘"味消失
- τ 适中（0.2–0.6s）：phugoid 明显但不发散，飞行有"飘"的手感
- τ 很大（>5s）：实质关闭，固定姿态行为

## 9. 终止条件与结算

### 9.1 终止条件

- **触地**：`p.z ≤ 0` → 强制 `p.z = 0`，仿真结束
- **超时**：`t > 30s` → 安全兜底

### 9.2 结算字段

| 字段 | 公式 | 物理含义 |
|---|---|---|
| `distance` | `p.y`（最终位置的世界 Y 坐标） | 沿基准线（+Y 射线）的飞行距离 |
| `lateral` | `p.x`（最终位置的世界 X 坐标） | 横向偏移，+ 为右偏 |
| `maxAlt` | `max(p.z)` 实时追踪 | 飞行最高点 |
| `time` | 仿真累计时间 | 总飞行时长 |
| `v0` | 出手初速 | — |
| `pitch` | 出手俯仰角 | — |
| `roll` | 出手横滚角 | — |

**距离语义**：基准线是从原点沿 +Y 方向延伸的射线。落地点 `(x, y, 0)` 到基准线的垂足为 `(0, y, 0)`，垂足在标尺上的刻度即 y 值。`lateral = x` 为带符号的垂线长度。

## 11. Cocos Creator 迁移要点

### 11.1 坐标系映射

Cocos Creator 3.x 默认世界系：**+X 右, +Y 上, -Z 前**（右手系，相机看向 -Z）。

本方案的坐标约定：**+X 右, +Y 前, +Z 上**。

从本方案世界系 `(x_w, y_w, z_w)` 到 Cocos 世界系 `(x_c, y_c, z_c)` 的映射：

```
x_c =  x_w       // 右 → 右
y_c =  z_w       // 上 → 上
z_c = -y_w       // 前 → -前（Cocos 的 -Z 是前方）
```

等价于绕世界 X 轴旋转 +90° 再绕世界 Y 轴旋转 180°。

**关键影响**：
- 重力在 Cocos 中为 `(0, -g, 0)` 而非本方案的 `(0, 0, -g)`
- 基准线（本方案的 +Y 轴）在 Cocos 中沿 -Z 方向
- `p.z`（本方案高度）→ Cocos 中的 `p.y`
- `p.y`（本方案距离）→ Cocos 中的 `-p.z`

**建议**：在仿真模块内部保持本方案的坐标约定（右手系，Z 为上），只在渲染层做坐标映射。这样物理代码完全不必修改。

### 11.2 传感器 API

| 功能 | 浏览器 API | 微信小游戏 API |
|---|---|---|
| 加速度计 | `DeviceMotionEvent` | `wx.onAccelerometerChange` |
| 陀螺仪 | `DeviceMotionEvent` | `wx.onGyroscopeChange` |
| 频率设置 | 不可控（系统决定） | `wx.startAccelerometer({ interval: 'game' })` |
| 振动 | `navigator.vibrate` | `wx.vibrateShort` |

微信小游戏返回的加速度计/陀螺仪数据轴定义与 W3C 标准基本一致，但需要在小规模真机测试时验证各轴的符号映射。

### 11.3 数值计算

- Cocos Creator 提供 `Vec3`、`Quat` 等数学库，可直接替代本方案的 V/Q 工具函数
- Cocos `Quat.integrate` 或手动指数映射均可用于四元数积分
- 气动力计算纯属自定义，不依赖 Cocos 物理引擎

### 11.4 渲染与物理分离

物理仿真模块（Simulator）应完全独立于渲染：
- 每帧 `step(dt)` 推进仿真状态
- 渲染层读取 `p, v, fwd, up, traj` 用于 3D 显示
- HUD 读取 `p.y, p.z, maxAlt, t, speed, alpha` 等用于 UI

### 11.5 微信小游戏特有注意事项

- **后台暂停**：`wx.onHide` 时暂停传感器和仿真，`wx.onShow` 时恢复
- **性能**：子步计算量可控（最大 64 子步/帧 × 约 50 次浮点运算 = ~3200 次/帧），远低于微信小游戏的 CPU 预算
- **音频**：使用 `wx.createInnerAudioContext` 替代 WebAudio API
- **振动**：使用 `wx.vibrateShort` 替代 `navigator.vibrate`
